# Mission

## Why Kresh exists

Modern AI systems are powerful, but intelligence itself is fragmented.

Skills, workflows, design systems, engineering practices, agent configurations, writing styles, and personalities are scattered across prompts, GitHub repositories, Reddit threads, documentation, and personal notes.

There is no standard way to package, share, and reuse them.

Software became powerful because code became modular.

Intelligence should be modular too.

---

## Our mission

Kresh exists to make intelligence installable.

We are building an open registry where developers can package, share, and compose intelligence modules for AI systems.

---

## What we are building

Kresh is an open registry for:

* Skills
* Agents
* Workflows
* Design systems
* Thinking frameworks
* Writing styles
* Engineering practices
* Personalities

These modules can be installed individually or combined together to create more capable AI systems.

---

## Who Kresh is for

Kresh is built primarily for:

* Developers
* Product engineers
* AI builders
* Designers
* Researchers
* Students

---

## What Kresh is not

Kresh is not:

* A social network
* A roleplay platform
* A chatbot
* A prompt marketplace

Our goal is to build infrastructure for reusable intelligence.

---

## Long-term vision

Just as npm became the registry for software packages and Docker became the standard for containers, Kresh aims to become the registry for intelligence.

Future AI systems, agents, IDEs, and applications should be able to install intelligence as easily as developers install packages today.

```bash
kresh install system-design
kresh install apple-design
kresh install researcher
kresh install senior-engineer

kresh agent add cursor-pro
```

Intelligence should be open, composable, and developer-friendly.
