# MISSION.md

## Kresh Mission

Kresh exists to make AI-native software development predictable, reusable, and maintainable.

Modern coding is no longer just human-to-code.
It is human + AI + repo knowledge + tools + workflows + validation.

The problem is that most projects still do not treat that knowledge as a first-class asset.
Instructions live in chats.
Conventions live in people’s heads.
Workflows are repeated manually.
Agent behavior changes from tool to tool.
Important context gets lost every time a session resets.

Kresh exists to fix that.

---

## The Problem Kresh Solves

### 1. AI forgets the project too easily

AI coding systems are powerful, but they are not automatically grounded in your project’s real rules.
Sessions reset.
Context gets dropped.
What was obvious in one chat is gone in the next.

Kresh makes project knowledge persistent.

It puts the important stuff in files the repo can carry everywhere:
- AGENTS.md
- CLAUDE.md
- SKILLS/
- MISSION.md
- RULES.md
- ARCHITECTURE.md
- WORKFLOWS.md

---

### 2. Instructions are scattered and inconsistent

Today, one tool wants one format.
Another tool wants another.
One team writes prompts in chat.
Another writes notes in README.
Another keeps everything in Notion, docs, comments, or memory.

That fragmentation creates drift.

Kresh gives the project a unified knowledge layer so humans and agents can read the same source of truth.

---

### 3. Teams depend too much on tribal knowledge

Most real project knowledge is not in code.
It is in the stuff people assume others “just know”:
- how to run the project
- what not to break
- what style to follow
- how to test changes
- which files matter
- which patterns are approved
- which shortcuts are dangerous

That is a weak system.
Weak systems break when the one person who knows everything is unavailable.

Kresh turns tribal knowledge into documented system knowledge.

---

### 4. AI agents are too generic by default

A generic agent is useful, but not enough.

A serious project needs an agent that understands:
- the architecture
- the stack
- the repo structure
- naming conventions
- security rules
- deployment flow
- quality standards
- domain-specific behavior

Kresh is built so agents are not guessing.
They are reading project-specific instructions and skills before acting.

---

### 5. Repeated tasks waste time because skills are not modular

Teams keep re-explaining the same work:
- testing rules
- commit conventions
- code review standards
- build steps
- release flow
- data formatting
- debugging patterns

That repetition is stupid.
The machine should remember the procedure, not the person repeating it.

Kresh treats skills as reusable modules.
A skill should be something you define once, version once, improve once, and reuse everywhere.

---

### 6. Tooling is expanding faster than coordination

The ecosystem now includes repo instructions, custom agents, skills, context protocols, tool calls, and orchestration layers.
That is good, but it also creates chaos if every layer speaks a slightly different language.

Kresh exists to reduce that chaos.

It is not just documentation.
It is coordination infrastructure.

---

## What Kresh Is

Kresh is a documentation-first framework for AI-assisted development.

It helps a repository describe:
- what the project is
- how the project works
- how agents should behave
- what skills exist
- how tasks should be executed
- how outputs should be validated

Kresh is designed to make projects self-explaining.

A new human should be able to enter the repo and understand it.
A new AI agent should be able to enter the repo and act correctly.

That is the standard.

---

## Core Mission

Build the missing layer between code and intelligence.

Kresh exists to turn repositories into structured operating systems for humans and AI agents, where knowledge is explicit, workflows are repeatable, and project behavior is predictable.

---

## Principles

### Knowledge must live with the code

If the codebase depends on a rule, that rule should be documented beside the code.

Not in memory.
Not in chat history.
Not in someone’s head.

---

### Instructions must be local

Project behavior should be defined close to the project itself.

A repo should carry its own expectations, so the right behavior travels with the code wherever it goes.

---

### Skills should be reusable

A good skill is not a one-off prompt.
It is a portable capability with clear inputs, outputs, and usage rules.

---

### Agents should be constrained, not worshipped

AI should be powerful, but not free to invent project policy.

The repo defines the boundary.
The agent works inside it.

---

### Validation matters

Good output is not enough.
Output must be checked against reality.

That means build checks, tests, reviews, and explicit success criteria are part of the system.

---

### Standards beat improvisation

A project should not depend on whoever is online that day.

It should depend on documented standards that any competent human or agent can follow.

---

## Vision

Kresh aims to become an open, practical standard for AI-native project knowledge.

Not a chat wrapper.
Not a personality layer.
Not another prompt dump.

A real operating layer for collaboration.

A place where:
- AGENTS.md defines agent behavior
- CLAUDE.md preserves persistent project context
- SKILLS define reusable capabilities
- MISSION.md defines purpose
- supporting docs define rules, architecture, and workflows

Together, these files create a system that is understandable, maintainable, and scalable.

---

## Long-Term Goal

Make every serious repository self-describing for both humans and AI.

If a project is well-built, then:
- the rules are written down
- the workflows are repeatable
- the skills are modular
- the agent behavior is explicit
- the knowledge survives time
- the project does not rot when context changes

That is what Kresh is for.
